# Contributing to JobFlow

First off, thank you for considering contributing to JobFlow. It's people like you that make JobFlow such a great tool.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for JobFlow.
* Make sure you have the latest version.
* Check if the issue has already been reported.
* Use the provided Bug Report template when opening an issue. Include detailed steps to reproduce the bug.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion for JobFlow, including completely new features and minor improvements to existing functionality.
* Use the Feature Request template to create an issue. 
* Detail the purpose, behavior, and potential impact of the feature.

### Your First Code Contribution
1. **Fork the repository** on GitHub and clone it locally.
2. **Create a branch** for your edits (`git checkout -b feature/my-feature` or `git checkout -b bugfix/issue-description`).
3. Set up the local environment by following the `README.md` instructions.
4. **Make sure your code lints and is properly formatted.** This project uses standard formatting guidelines. Keep the style consistent with the rest of the project.
5. Create a Pull Request using the PR Template. Link the PR to the issue if applicable.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the `README.md` and documentation with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations, and container parameters.
3. You may merge the Pull Request once you have the sign-off of at least one other developer, or if you do not have permission to do that, you may request the reviewer to merge it for you.

## Development Setup

Please refer to the `README.md` file in the root of the repository for detailed setup and installation instructions for both the frontend and backend.

## Styleguides

### Git Commit Messages
* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit the first line to 72 characters or less
* Reference issues and pull requests liberally after the first line
